Even older version: there is one big C file with translated code and two small auxilary files.

Sources ( do not click, download them):
  • tex.c(510 kB) - the main C file
  • funcs.c, inipool.c, pool_str.c - auxillary files
  • p2c/p2c.h - header file with translated macros
  • Makefile
The compilation works using 'gcc' on Linux. The sources are pretty generic ANSI C, however some functions are about 1000 lines long (and tex.c is big) so they my extend compiler limits. The executable ('ttex') is an INITEX -- you may just input 'plain' in your file or first generate a format and then use it.

Usage:
  • ttex file.tex
  • ttex '&format' file.tex
In the first case 'file.tex' must input the format file (source). File handling is primitive: set TEXINPUTS, TEXFONTS and TEXFORMATS to to specify path (list of directories) where the executable looks for input files, tfm-s and formats respectively.

BUG: If file has no '\end' the program will loop endlessly.
